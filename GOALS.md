# 목표 명확성 개선 계획 (Goal Clarity)

## 문제 정의

현재 게임은 **끝없는 웨이브 생존**이다. 좀비를 처치하고, 5웨이브마다 보스가 나오고, 점수가 쌓이지만 —
플레이어에게 **"어디까지 하면 되는가 / 무엇이 성공인가"** 가 제시되지 않는다. 그래서:

- 시작하면 무엇을 향해 달리는지 모른다(막연히 오래 버티기).
- "클리어"라는 성취 순간이 없다 — 죽으면 그냥 게임오버.
- 점수는 남지만 그 점수가 "좋은지" 판단할 기준(목표선)이 없다.

핵심 원인은 **매크로 목표(이 판의 결승선)와 마이크로 목표(지금 당장 할 일)가 둘 다 부재**하다는 것.

## 설계 원칙 — 3층 목표 구조

| 층위 | 질문 | 해답(제안) |
| --- | --- | --- |
| **마이크로** (지금) | "지금 뭘 해야 해?" | 항상 보이는 다음 목표: "웨이브 3/10 — 보스까지 2웨이브" |
| **매크로** (이 판) | "언제 이기는 거야?" | 명시적 결승선: "10웨이브 도달 + 보스 처치 = 스테이지 클리어" |
| **메타** (반복 동기) | "그다음엔?" | 클리어 후 엔들리스(점수 경쟁) + 난이도별 클리어 배지 |

---

## 개선안 상세

### 1. 명시적 승리 조건 — "스테이지 클리어" (매크로) ★핵심
끝없음을 유지하되, **분명한 결승선 하나**를 만든다.
- 예: **10웨이브의 보스(2번째 보스)를 처치하면 "STAGE CLEAR"**. (난이도별로 목표 웨이브를 다르게 둘 수 있음: Easy 8 / Normal 10 / Hard 12)
- 클리어 시 **게임오버와 확실히 구분되는 승리 화면**(금색 팡파르 + "STAGE CLEAR" + 통계 + 별점).
- 클리어 후: **"엔들리스로 계속"** 버튼 → 이후는 점수 러시(이미 6웨이브+ 무한 스케일링이 들어가 있어 자연 연결).
- 데이터: `RankingManager` 에 클리어 여부/클리어 웨이브를 기록, 메뉴에 **난이도별 클리어 배지**.

### 2. 항상 보이는 "다음 목표" 배너 (마이크로) ★핵심·저비용
HUD 상단에 현재 목표 한 줄을 상시 노출한다. 상태에 따라 문구가 바뀐다:
- 일반 웨이브: **"웨이브 3 — 좀비 처치 (27/60)"** (이미 있는 진행 바 + 숫자 활용)
- 보스 임박: **"⚠ 다음 웨이브: 보스"**
- 보스전: **"보스를 처치하라!"**
- 목표 웨이브 근접: **"결승선까지 2웨이브"**
→ 기존 `Events.wave_changed` / `wave_progress_changed` / `boss_spawned` 시그널로 문구만 갱신하면 됨.

### 3. 보스를 "체크포인트/장(章)"으로 승격 (마이크로→매크로 연결)
- 보스 등장 전 **"BOSS APPROACHING"** 풀스크린 경고 연출(짧게) + 전용 사운드.
- 보스에게 **이름** 부여(1번 보스: "The Horde Alpha" 등) → 이정표가 기억에 남는다.
- 보스 처치 = **장 클리어 스탬프**("Chapter 1 Clear"). 다음 목표로 자연 유도.

### 4. 진행/목표 시각화 (마이크로)
- HUD 웨이브 진행 바(이미 추가됨) 위에 **전체 스테이지 진행 눈금**(예: ●─●─●─★ 로 보스/결승선 표시).
- 결승선 웨이브를 눈금 끝의 **★(깃발)** 으로 표시 → "여기까지 가면 끝"이 한눈에.

### 5. 최초 진입 온보딩 (마이크로)
- 첫 플레이(세이브 없음) 시, 시작 직후 3초 콜아웃: **"목표: 10웨이브의 보스를 처치하라"**.
- 조작 힌트(조이스틱=이동, 자동 발사)도 이때 1회.

### 6. 결과 화면 목표 대비 피드백 (메타)
- 게임오버 시 **"목표까지 3웨이브 남았습니다"** 처럼 결승선 대비 얼마나 왔는지 표시 → 다음 판 동기 부여.
- 클리어 시 별점(클리어 웨이브·남은 체력·소요 시간 기준 1~3★).

## 구현 로드맵 (우선순위)

### P0 — 즉시 체감, 저비용 (하루 이내)
1. **다음 목표 배너**(HUD 한 줄, 시그널 문구 매핑). — #2
2. **보스 임박 경고** 텍스트 + 사운드(이미 있는 `boss_spawned` 활용). — #3
3. **결승선 웨이브 상수**(`Events` 에 `goal_wave` 난이도별) + 게임오버 화면에 "목표까지 N웨이브". — #1·#6
   → Locale 키 추가(en/ko/ja).

### P1 — 승리 루프 완성 (2~3일)
4. **STAGE CLEAR 승리 화면**(게임오버와 분리된 별도 패널 + 팡파르 + 별점). — #1
5. **"엔들리스로 계속" / "메뉴"** 분기. — #1
6. **RankingManager 에 클리어 기록** + 메뉴 난이도 버튼에 **클리어 배지**. — #1

### P2 — 몰입/리텐션 (선택)
7. 보스 이름·풀스크린 인트로 연출. — #3
8. 스테이지 진행 눈금(●─●─★) HUD. — #4
9. 최초 진입 온보딩 콜아웃. — #5
10. 클리어 별점 산정·표시. — #6

## 코드 훅(구현 시 참고)

- 결승선/목표 웨이브: `Events.gd` 에 `goal_wave()`(난이도별 8/10/12) 추가.
- 승리 판정: `ZombieSpawner._wave_complete()` 에서 `_wave_num == goal_wave` 이고 보스 처치 시
  `Events.stage_cleared.emit()` (신규 시그널). 아니면 기존 흐름 유지.
- 승리 화면: `HUD.gd` 에 게임오버 패널과 형제로 `stage_clear_panel` 추가, `stage_cleared` 구독.
- 다음 목표 배너: `HUD.gd` 가 `wave_changed`/`boss_spawned`/`wave_progress_changed` 구독해 한 줄 갱신.
- 클리어 기록/배지: `RankingManager` 에 `mark_cleared(mode_id, wave)` + `MainMenu` 난이도 버튼 배지.
- 문구: `Locale.STRINGS` 에 `goal_banner_*`, `boss_incoming`, `stage_clear`, `go_to_goal_fmt` 등.

## 성공 기준(무엇이 나아졌다고 볼까)

- 플레이어가 시작 10초 안에 "이 판의 목표"를 말할 수 있다(=배너/온보딩 노출).
- "게임오버" 외에 **"클리어"라는 종료 상태**가 존재한다.
- 게임오버 시 결승선 대비 위치를 알아 **다음 판을 시작할 이유**가 생긴다.
- 난이도별 클리어 배지로 **재도전(상위 난이도) 동기**가 생긴다.

> 이 문서는 계획이다. P0 3개는 기존 시그널만으로 구현 가능한 저비용 항목이라, 승인하시면 바로 착수해
> "다음 목표 배너 + 보스 경고 + 결승선/게임오버 목표 표시"부터 반영하겠습니다.
