# 서비스 출시 체크리스트 (Launch Readiness)

게임을 "만든 것"에서 "서비스하는 것"으로 넘어가기 위한 남은 작업 정리.
현재는 **웹 빌드(GitHub Pages) + 스텁 광고 + 로컬 저장/랭킹** 상태 — 아래는 이를 실서비스로
끌어올리기 위한 항목이다. **A(플랫폼 결정)가 나머지 대부분을 좌우**하므로 먼저 확정해야 한다.

---

## A. 플랫폼 방향 결정 ⭐ (선행 — 모든 것의 분기점)
- **웹**(itch.io/포털/자체 도메인) vs **모바일 스토어**(Google Play / App Store) vs **둘 다**.
- 이 결정이 광고·결제·랭킹·저장·법무 방식을 전부 바꾼다.
  - 웹: AdMob 사용 불가(H5 광고망/자체 광고), 결제는 웹결제(Stripe 등), 배포는 정적 호스팅.
  - 모바일: Godot Android/iOS export + 스토어 심사 + Play Billing/StoreKit + PGS/Game Center.

## B. 수익화 (Monetization)
- [ ] **보상형 광고 실연동** — `AdManager.USE_STUB=false` + 실 광고 단위 ID. (부활 훅은 이미 존재)
  - 모바일: AdMob / AppLovin MAX / Unity LevelPlay 네이티브 플러그인.
  - 웹: 웹 리워드 광고 SDK(H5) 또는 네이티브 래핑. **AdMob 은 웹 미지원**.
- [ ] **광고 동의(UMP/CMP)** — GDPR/CCPA 지역 동의 팝업(광고 SDK 필수 요구).
- [ ] **인앱결제(IAP)** — 광고 제거, 골드 팩, 스킨 등. Play Billing / StoreKit / 웹 결제.
- [ ] 결제 복원(Restore), 영수증 검증(서버), 환불 처리.

## C. 데이터 · 백엔드
- [ ] **클라우드 세이브** — 현재 `user://` 로컬(웹=IndexedDB, 데이터 삭제 시 소실).
      PGS Saved Games / Game Center / 자체 서버 / Firebase 중 택1. 기기 변경 이어하기.
- [ ] **온라인 글로벌 랭킹** — 현재 로컬만(`LocalRankingBackend`). 웹 글로벌 리더보드는
      서버(Firebase/자체 API) 필요. **치팅 방지**(서버 점수 검증·리플레이 해시) 설계.
- [ ] **계정** — 게스트 플레이 + 선택적 로그인(Google/Apple). 데이터 이관.
- [ ] 세이브 **버전 마이그레이션** — 스키마 변경 시 구세이브 자동 변환(현재 단순 JSON).

## D. 법무 · 컴플라이언스 (스토어·광고 시 필수)
- [ ] **개인정보 처리방침 · 이용약관** URL (광고/분석 쓰면 반드시).
- [ ] **연령 등급**(IARC) + 스토어 **데이터 안전(Data Safety)** 섹션 작성.
- [ ] 아동 대상 여부 판정 → COPPA / Google 가족정책 대응(해당 시 광고 제한).
- [ ] 오픈소스 라이선스 고지(에셋 CC0 출처·폰트 라이선스 등).

## E. 분석 · 운영 (LiveOps)
- [ ] **분석/텔레메트리** — 현재 전무. 최소: 리텐션(D1/D7), 웨이브 도달 분포, 이탈 지점,
      결제 퍼널, 광고 시청률. Firebase Analytics / GA4.
- [ ] **크래시·에러 리포팅** — Crashlytics / Sentry.
- [ ] **원격 설정(Remote Config)** — 밸런스 수치·광고 빈도를 배포 없이 조정
      (이미 `build_info` 스탬프로 라이브 빌드 식별은 됨).

## F. 품질 · 안정성
- [ ] **실기기 테스트** — 저사양 안드로이드 포함, 다양한 해상도·노치·세이프에어리어.
- [ ] **웹 오디오 언락** — 첫 터치 후 재생 정책 재확인(모바일 브라우저).
- [ ] 네트워크 예외/오프라인 처리(광고·랭킹 실패 시 graceful).
- [ ] 성능 예산 — 최대 동시 좀비/투사체/FX 상한 스트레스 테스트(웹 저사양).
- [ ] 세이브 손상 복구(부분 파싱 실패 시 기본값).

## G. 스토어 · 출시 자산 (모바일 진행 시)
- [ ] Godot **Android/iOS export 프리셋** — 패키지명, 최소 SDK, **권한 최소화**, 키스토어 서명.
- [ ] 앱 아이콘 · 스크린샷(기기별) · 프로모 영상 · 스토어 설명(다국어 en/ko/ja).
- [ ] 스토어 심사 정책 점검(광고 정책·개인정보·콘텐츠 등급).

## H. 콘텐츠 · 리텐션 (장기)
- [ ] **밸런스 실측 튜닝**(BOSS_PLAN P5 / BALANCE) — 실플레이 데이터 기반.
- [ ] 온보딩/튜토리얼 강화(이미 `IntroStory` 존재), 목표 명확화(GOALS).
- [ ] 리텐션 루프 — 일일 보상, 도전과제/업적, 시즌/이벤트, 신규 보스·무기 파이프라인.
- [ ] 푸시 알림(모바일) — 복귀 유도.

---

## 권장 우선순위 (웹 우선 가정 시)
1. **A 플랫폼 확정** → 2. **분석·크래시 리포팅(E)**: 뭐가 문제인지 봐야 개선 가능 →
3. **클라우드 세이브·온라인 랭킹(C)**: 유지 동기 → 4. **광고/결제 실연동(B)** + **법무(D)** →
5. **품질·스토어 자산(F/G)** → 6. **리텐션 콘텐츠(H)**.

> 분석이 가장 저비용·고효과 선행 작업이다 — 데이터 없이 밸런스·수익화를 조정하는 건 추측이다.
