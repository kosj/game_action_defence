# 보스 다양화 계획 (Boss Variety Plan)

웨이브 5의 배수마다 등장하는 단일 근접 보스를, **고유 행동 패턴을 가진 여러 아키타입**으로
확장한다. "총을 쏘는 보스"를 필두로, 각 보스가 서로 다른 위협·대응법을 요구하게 만든다.

---

## 1. 현재 상태 (as-is)

- **보스는 1종뿐**: `Boss.gd` = 플레이어를 향해 직진 돌격하는 근접형(Brute). 접촉 피해만.
- 등장: `ZombieSpawner._spawn_boss()` — 5웨이브마다, `boss_count = wave / 5` (1, 2, 3…)로 스탯 스케일.
- `boss.setup({max_health, speed, contact_damage, score, gold})` 로 스탯 주입.
- 공용 인프라(재사용 가능):
  - **`EnemyBullet` (scene+script)** — 적 투사체. 이미 스피터 좀비가 사용. 거리 기반 명중 → `player.take_hit()`.
  - **`Zombie.gd` 행동 분기 패턴** — `weaver/spitter/bomber` 를 `match behavior:` 로 분기. 보스도 이 방식이 어울림.
  - **`FXBurst`** — 확산 파동 연출(등장·피격·사망). start_delay 지원.
  - **HUD 보스 체력바** — `Events.boss_spawned/boss_health_changed/boss_died` 시그널로 이미 연동.
  - **`Pool`** — 투사체/미니언 재사용.

즉, 새 시스템을 밑바닥부터 만들 필요 없이 **기존 조각을 조합**하면 된다.

---

## 2. 목표 (to-be)

1. 보스 **아키타입 5종** (근접/사격/소환/포격/광폭) — 각기 다른 대응을 강제.
2. **총 쏘는 보스**(사격형)를 우선 구현 — 사용자 요청의 핵심.
3. 보스마다 **텔레그래프(예비 동작)** 를 넣어, 모바일에서도 보고 피할 수 있는 공정한 난이도.
4. 보스별 **HP 임계 페이즈(격노)** — 체력이 낮아지면 공격이 격화(선택적, 후반 보스부터).
5. HUD 에 **보스 이름/타입 표시**(선택) — "지금 무슨 보스인지" 인지.

---

## 3. 아키타입 로스터

| # | 이름(가칭) | 행동 | 위협 | 대응법 | 재사용 |
|---|---|---|---|---|---|
| A | **브루트 Brute** | 현재 근접 돌격 | 접촉 피해·높은 HP | 카이팅(원거리 딜) | 현행 그대로 |
| B | **거너 Gunner** ⭐ | 거리 유지하며 조준 사격(3발 스프레드 / 방사형 난사) | 원거리 투사체 | 엄폐 없이 좌우 회피, 빠른 접근전 강요 | `EnemyBullet` 그대로 |
| C | **서머너 Summoner** | 주기적으로 호위 좀비 소환 | 잡몹 누적 압박 | 소환 즉시 정리 or 보스 폭딜 | `ZombieSpawner._spawn_one` 재사용 |
| D | **바머 Bomber/포격형** | 플레이어 예측 위치에 지연 폭발 탄 투하(바닥 표식→폭발) | 광역 지연 피해 | 표식 뜨면 이동 | `FXBurst` + 신규 지면 인디케이터 |
| E | **버서커 Berserker/돌진형** | 느린 추적 ↔ 텔레그래프 후 초고속 대시 반복 | 순간 돌진 접촉 | 대시 방향 예측·측면 회피 | 상태머신만 신규 |

⭐ = 우선 구현. B(거너)가 "총 쏘는 보스" 요구를 직접 충족.

### 거너(B) 상세 — 우선 구현 대상
- 스피터 좀비 로직의 보스판. `SPIT_KEEP_DIST` 처럼 **유지 거리**를 두고 카이팅.
- 발사 패턴 3단계(회차/페이즈에 따라 격화):
  1. **단발 조준** — 플레이어 방향 1발.
  2. **3발 스프레드** — ±15° 부채꼴.
  3. **방사형 난사** — 8~12발 원형(페이즈2/격노 시).
- 발사 전 **총구 점멸(0.3~0.4s 텔레그래프)** — `body.modulate` 깜빡 + 조준선. 보고 피할 여지.
- HP 는 브루트보다 낮게(원거리라 안전하므로), 접촉 피해도 낮게.

## 4. 아키텍처 설계

`Zombie.gd` 의 행동 분기 방식을 그대로 따른다(코드베이스 일관성).

### 4.1 보스 정의 테이블 (`ZombieSpawner.gd` 또는 신규 `BossDB.gd`)
```gdscript
# 예시 스케치
const BOSS_TYPES := [
    {"key":"brute",    "archetype":"melee",   "hp_mul":1.00, "speed":1.00, "contact":2, "tint":Color(0.55,0.12,0.14)},
    {"key":"gunner",   "archetype":"gunner",  "hp_mul":0.80, "speed":0.85, "contact":1, "tint":Color(0.20,0.35,0.65)},
    {"key":"summoner", "archetype":"summoner","hp_mul":0.90, "speed":0.70, "contact":2, "tint":Color(0.30,0.55,0.25)},
    {"key":"bomber",   "archetype":"bomber",  "hp_mul":0.95, "speed":0.75, "contact":1, "tint":Color(0.60,0.40,0.15)},
    {"key":"berserker","archetype":"berserk", "hp_mul":1.05, "speed":1.00, "contact":3, "tint":Color(0.60,0.15,0.35)},
]
```
- 기존 `_spawn_boss()` 의 스탯 계산(HP/score/gold 스케일)은 유지하되, `hp_mul`·`archetype`·`tint` 를 추가 주입.

### 4.2 `Boss.gd` 확장
- `setup()` 에 `archetype`, `tint` 수신. `body.modulate = tint` 로 종별 색 구분.
- `_physics_process` 에 `match _archetype:` 분기 → `_behave_melee/_behave_gunner/...`.
- 공용 유지: 등장 연출·HP바·`take_damage`·`_die`(보상/충격파). **파괴 안정성**(루트 scale 애니 금지) 규칙 준수.
- 발사/소환/포격 헬퍼는 각 `_behave_*` 안에서 쿨다운 타이머로 구동(스피터와 동일 구조).

### 4.3 텔레그래프 유틸
- 공용 `_telegraph(duration, color)` — 발사/대시/투하 직전 `body.modulate` 점멸 + `_draw` 에 조준/경고 그리기.
- 포격형(D)은 **지면 원형 인디케이터**가 필요 → `FXBurst` 를 "경고 링(피해 없음) → 지연 후 폭발 링(피해)" 2단으로 재사용하거나, 간단한 전용 `GroundWarning` Node2D 신규.

### 4.4 보스 선택 규칙
- **초반 고정 순서**로 학습시키고 이후 무작위(반복 회피):
  - 1차 보스(W5) = Brute, 2차(W10) = Gunner, 3차(W15) = Summoner, 4차(W20) = Bomber, 5차(W25) = Berserker.
  - 6차 이후 = 이전과 다른 타입 가중 무작위.
- 난이도별 조정: Hard 는 첫 등장부터 페이즈2 패턴 허용 등(BALANCE 와 연동).

### 4.5 HUD (선택)
- 보스 체력바 위/옆에 **보스 이름 라벨**(Locale 키). `boss_spawned` 시그널에 `name` 추가하거나 별도 시그널.

## 5. 단계별 로드맵 (각 단계 독립 배포 가능)

- **P0 — 리팩터 기반 (선행)**: `Boss.gd` 에 `archetype` 분기 골격 + 보스 정의 테이블 도입. 기존 Brute 를 `melee` 아키타입으로 이관(동작 100% 동일). 회귀 없음 확인.
- **P1 — 거너(총 쏘는 보스) ⭐**: `_behave_gunner` + 텔레그래프 + 3발 스프레드. 2차 보스(W10)에 배치. → **사용자 요청 핵심 완료**.
- **P2 — 서머너**: 주기적 호위 소환. 소환 상한/간격 밸런스.
- **P3 — 바머(포격형)**: 지면 경고 인디케이터 + 지연 폭발.
- **P4 — 버서커(돌진형)**: 대시 상태머신 + 텔레그래프.
- **P5 — 페이즈/격노 + HUD 이름 + 밸런스 패스**: HP 임계 격화, 보스 선택 규칙 확정, 난이도별 튜닝, `BALANCE.md` 갱신.

각 단계는 빌드→배포→실플레이 확인 후 다음 단계로.

---

## 6. 밸런스 원칙

- **원거리 보스는 HP↓·접촉↓**, 근접 보스는 HP↑. 위협 총량을 균형.
- 모든 특수 공격에 **가시적 텔레그래프**(≥0.3s) — 모바일 세로 화면·터치 조작 기준 회피 가능해야 공정.
- 투사체 속도는 플레이어 이속(220) 대비 회피 여지 있게(스피터 기준 참고, 보스는 약간 빠르게).
- 보상(score/gold)은 위협도에 비례 상향. 모드별 랭킹 분리는 유지.
- 프레임 예산: 투사체·미니언은 반드시 `Pool` 재사용, 방사형 난사도 상한 두기.

---

## 7. 리스크 & 주의

- **파괴 안정성**: 루트 `CharacterBody2D` 의 scale/위치를 트윈하면 `move_and_slide` 가 깨져 보스가 얼어붙는다(기존 주석 규칙). Body 스프라이트만 애니메이트.
- **난사 성능**: 방사형 다발 + 저사양 웹. 투사체 수 상한·풀링 필수.
- **텔레그래프 누락 = 불공정**: 순간사·즉발 폭발 금지. 반드시 예비 동작.
- **가독성**: 종별 색(tint)·이름으로 구분. 잡몹과 헷갈리지 않게 크기/오라 유지.
